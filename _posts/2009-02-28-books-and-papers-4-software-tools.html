--- layout: post title: Books and Papers - 6 - Software Tools, Kernighan and Plauger date: '2009-02-28T06:59:00.008Z' author: John Overington tags: - Programming - Books and Papers modified_time: '2009-03-26T06:04:46.764Z' thumbnail: http://2.bp.blogspot.com/_SlPG9DIz_AA/SajkueDKzRI/AAAAAAAAAKE/cxlgTs2wq8g/s72-c/soft.png blogger_id: tag:blogger.com,1999:blog-2546008714740235720.post-6050405672519339732 blogger_orig_url: http://chembl.blogspot.com/2009/02/books-and-papers-4-software-tools.html --- This was the first proper programming book that I studied, and it is an old one too, written before the web, web-services, and networking - good old fashioned UNIX programming computation. Few words, lots of concepts, and merits revisiting now and then - the prose in Kernighan's books is just excellent, perfect pace, and combines advice with examples of little code snippets. Probably the best thing about the book (for me) is the use of ratfor - a derivative of Fortran that looks a lot like C, and has some of the best elements of both languages. It is also incredibly quick to code in. I have just downloaded some updated versions of ratfor for my mac, so expect some pretty unusual looking Open Source protein analysis tools anyday soon!

Anyway, although ratfor is used, for almost everyone, the code there will just be pseudocode for implementation and inspiration of code in a more fashionable language. If anything, using a non-current language forces thinking more deeply about the actual structure of the program.
